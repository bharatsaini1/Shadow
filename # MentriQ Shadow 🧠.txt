## What We Are Building

MentriQ Shadow solves one critical problem: **students graduate with theory but zero real-world experience**. Companies want experienced freshers. Freshers can't get experience without a job. This is the chicken-and-egg trap.

**MentriQ Shadow breaks this loop** by simulating a full work environment powered by AI:

| Traditional EdTech | MentriQ Shadow |
|---|---|
| Watch video → take quiz | Work a real task → get AI reviewed |
| Passive learning | Active simulation |
| Certificate of completion | Verifiable performance report (Shadow Passport) |
| Generic content | Role-specific AI simulation |
| No team interaction | AI teammates, manager, client |

The platform is built in **3 phases**:

- **Phase 1** — Build and test all AI pipelines as Jupyter Notebooks
- **Phase 2** — Convert notebooks into a Django + Django REST Framework backend
- **Phase 3** — Build the frontend using **Next.js 14 (App Router) + Tailwind CSS**

### 🎨 Design Direction — READ THIS BEFORE WRITING ANY CODE

MentriQ Shadow is not an edtech app, not a quiz platform, not a dark-mode Coursera.

**It is a company. Students are employees. The UI is their office.**

Everything — every font choice, every hover state, every timing curve — must make the student feel like they are already at work. Not "learning how to work." Actually working.

---

### The Aesthetic Reference

The design direction is **Editorial Brutalism meets Precision Tooling**.

Think: The Economist's typographic discipline + Linear's density + a paper invoice that got digitized by a very good designer. Not a startup. Not a "product." A place where serious work happens.

**The three principles:**

1. **Type-forward** — Typography carries the hierarchy. Color does not. A page should read clearly in grayscale.

2. **Controlled tension** — The layout is rigid and grid-based but with deliberate moments of asymmetry. One element breaks the grid per screen. That one thing is what the user remembers.

3. **Signal over noise** — Every color usage means something. Red means overdue. Green means success. Amber means warning. Blue means active. Nothing is colored for decoration.

**What this replaces from the old design:**
- "Dense. Dark. Alive." was the right instinct but produced generic dark-SaaS. We are going further.
- The old approach used blue everywhere (primary actions, active states, borders, badges). Now blue is reserved only for the single most important interactive element on a given screen.
- Glassmorphism, glow effects, and gradient meshes are removed. Replaced with: paper-like surfaces, strong editorial typography, and precise micro-contrast.

### 🛡️ Route Protection

**`components/layout/ProtectedRoute.tsx`:**

```typescript
"use client";
import { useAuth } from "@/lib/hooks/useAuth";
import { useRouter } from "next/navigation";
import { useEffect } from "react";
import Spinner from "@/components/ui/Spinner";

export default function ProtectedRoute({ children }: { children: React.ReactNode }) {
  const { user, loading } = useAuth();
  const router = useRouter();

  useEffect(() => {
    if (!loading && !user) router.push("/login");
  }, [user, loading, router]);

  if (loading || !user) return (
    <div className="flex h-screen items-center justify-center bg-ink">
      <Spinner size="lg" color="default" />
    </div>
  );

  return <>{children}</>;
}

  loading: () => <div className="skeleton h-full w-full" />,
});

export const EDITOR_OPTIONS = {
  theme: "vs-dark",
  fontSize: 13,
  fontFamily: "'IBM Plex Mono', monospace",
  fontLigatures: false,  // IBM Plex Mono does not have ligatures
  minimap: { enabled: false },
  scrollBeyondLastLine: false,
  wordWrap: "on" as const,
  automaticLayout: true,
  tabSize: 2,
  lineNumbersMinChars: 3,
  folding: false,
  renderLineHighlight: "gutter" as const,
  contextmenu: false,
  padding: { top: 12 },
  scrollbar: { vertical: 'hidden' as const, horizontal: 'hidden' as const },
};

interface CodeEditorProps {
  value: string;
  onChange: (value: string) => void;
  language?: string;
  height?: string;
}

export default function CodeEditor({ value, onChange, language = "javascript", height = "100%" }: CodeEditorProps) {
  return (
    <MonacoEditor
      height={height}
      language={language}
      value={value}
      onChange={(v) => onChange(v ?? "")}
      options={EDITOR_OPTIONS}
      loading={<div className="skeleton h-full w-full" />}
    />
  );
}
```

## Data Flow

```
Student selects career track
      ↓
POST /api/v1/simulations/start → session_id
      ↓
POST /api/v1/simulations/{id}/generate-day → Celery → RAG + Groq → Tasks saved
      ↓
Frontend polls GET /api/v1/celery-tasks/{id}/status/
      ↓
Student works in Simulation Workspace (3-column layout)
      ↓
POST /api/v1/tasks/{id}/submit → Celery evaluation task
      ↓
Celery: RAG rubrics → Groq reviewer → Evaluation saved
      ↓
Frontend polls GET /api/v1/tasks/{id}/evaluation → score + feedback
      ↓
Evaluation page: editorial score reveal
      ↓
POST /api/v1/simulations/{id}/team-message → Persona engine → Chat
      ↓
Student levels up → next day
```

## Critical Rules for OpenCode

**Follow these exactly. These prevent the "AI-generated look":**

1. **Font discipline** — Only use: Instrument Serif (display numbers + editorial headings), IBM Plex Sans (all UI text), IBM Plex Sans Condensed (badges, uppercase labels), IBM Plex Mono (code, IDs, timestamps, timers). Never use Inter, Roboto, or Geist. Never use Syne.

2. **Color discipline** — Signal blue (`#2563EB`) is used ONCE per screen, on the single primary CTA. Never use it for borders, badges, or decorative purposes. Semantic colors (go/caution/stop/mark) are used ONLY for their semantic meaning — never for variety.

3. **No glow effects** — Remove any `box-shadow` that uses colored values (rgba blue, rgba green, etc.). Shadows use only rgba(0,0,0,...). The only exception: focus rings (signal blue, opacity 0.4).

4. **No gradient text** — Never apply gradient via `bg-clip-text`. Score numbers and headings are solid `text-prose`.

5. **Borders are hairlines** — All borders use `border-rule` (1px solid #2A2A32) or `border-rule-2`. No colored borders except the active nav left accent and the "Most Popular" pricing card.

6. **Rounded corners are earned** — Default radius: `rounded-base` (6px) or `rounded-md` (8px). Only buttons use `rounded-pill`. Cards never exceed `rounded-lg` (12px).

7. **Type hierarchy through weight and size, not color** — The primary text hierarchy uses size and `font-medium` weight. Don't use color variation within the same text block to create hierarchy.

8. **Spacing is generous on the outside, tight on the inside** — Cards have 24px padding. Rows inside cards have 12px vertical gap. Sections between cards have 32px gap.

9. **The backend API is unchanged** — All field names, routes, and response formats match the Django backend exactly. Transform snake_case to camelCase only inside hooks (`lib/hooks/`).

10. **`<form>` tags never appear** — Use `<div>` wrappers and `onClick` handlers. This applies everywhere.

11. **Monaco Editor is always `next/dynamic` with `ssr: false`** — No static imports, ever.

12. **Skeleton loaders on every data component** — If `data === null`, render skeletons. Never render empty states with "No data found" text.
